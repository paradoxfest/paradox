Blind Coding Guidelines

  • Duration: 45 minutes
  • Task: Write functional code based on the given problem statement without access to documentation or auto-completion.
  • Tools: Only a basic code editor will be provided. Internet access is strictly prohibited.
  • Languages: Participants may code in any language they are comfortable with (pre-approved by organizers).
  • Evaluation Criteria:
    • Functionality: Does the code solve the problem as intended?
    • Efficiency: Is the code optimized and free of unnecessary logic?
    • Code Style: Clarity, readability, and structure of the code.
    • Adherence: Following the competition constraints without external help.
  • Participants: Up to 20 competitors will take part.
  • Scoring: Based on correctness, code quality, and speed of completion.

Note: Each participant will use an isolated machine for coding.
